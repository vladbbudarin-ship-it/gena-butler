# Исправление вложений

## 1. Название задачи
Исправить систему вложений в чате, кабинете Бударина и СУП.

## 2. Краткое описание задачи
Нужно было сделать безопасное открытие файлов через signed URL, добавить soft delete вложений, показать вложения в интерфейсе и не отправлять содержимое файлов в OpenAI.

## 3. Что было изменено
- Добавлен общий backend-helper для проверки доступа к вложениям.
- Добавлены функции получения signed URL и soft delete вложения.
- Добавлен общий React-компонент списка вложений.
- Чат пользователя, чат Бударина и кабинет Бударина теперь показывают вложения сообщений.
- В СУП файлы задач/проектов открываются через общий безопасный механизм.
- Для сообщений в чате добавлена загрузка файлов в private bucket `attachments`.
- Существующие файлы задач/проектов не публикуются напрямую, а открываются через signed URL.

## 5. Какие ошибки были найдены
- Для вложений не было общего безопасного способа открыть файл.
- В чате не было таблицы и UI-связки файла с конкретным сообщением.
- Для удаления файлов не было soft delete.
- Файлы задач/проектов отображались как статичный список без открытия и удаления.

## 6. Какие ошибки были исправлены
- Открытие файлов теперь идёт через Netlify Function и Supabase signed URL.
- Удаление вложений теперь меняет `deleted_at` / `deleted_by`, не удаляя файл физически.
- Удалённые файлы не возвращаются как активные в UI.
- Доступ к файлу проверяется по участию в чате, доступу к задаче/проекту или правам Бударина.

E2E-тесты не запускались в этой задаче.

## 10. Что нужно проверить вручную
- Выполнить SQL из `supabase/attachments-schema.sql` в Supabase SQL Editor.
- Проверить, что private bucket `attachments` создан.
- Отправить файл в чат с Будариным и открыть его.
- Отправить файл в direct chat и открыть его вторым участником.
- Удалить свой файл и убедиться, что сообщение осталось.
- Проверить файл задачи в СУП через кнопку открытия.

## 11. Риски и важные замечания
- До выполнения SQL новые chat-вложения не смогут сохраняться.
- Файлы задач/проектов используют существующий bucket `sup-project-files`; новые файлы сообщений используют bucket `attachments`.
- Содержимое файлов не передаётся в OpenAI.
- Физическое удаление из Storage не выполняется, используется soft delete.

## 12. Следующий рекомендуемый шаг
Выполнить SQL `supabase/attachments-schema.sql`, затем вручную проверить открытие и удаление вложений в чате и СУП.
