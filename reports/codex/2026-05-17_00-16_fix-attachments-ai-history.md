# Название задачи
Исправление вложений, AI-истории Пользователь+ и скрытия чата

# Краткое описание задачи
Проверить и точечно исправить работу файлов в чате и задачах СУП, отображение и удаление AI-истории, а также поведение кнопки “Скрыть” в кабинете Бударина.

# Что было изменено
- В чате пользователя и кабинете Бударина добавлена компактная кнопка “+” для выбора файлов рядом с кнопкой отправки.
- Разрешена отправка сообщения только с файлом без текста.
- Добавлен компактный preview выбранных файлов с размером и кнопкой удаления из выбора до отправки.
- В открытой задаче СУП добавлен отдельный блок “Файлы задачи”.
- Вкладка “Файлы” в проекте теперь показывает файлы выбранной задачи, а не смешивает их с проектными файлами.
- История AI-предложений проекта теперь загружается шире и фильтрует soft-deleted записи.
- В интерфейсе AI-истории добавлена компактная кнопка “Удалить”.
- Кнопка “Скрыть” в кабинете Бударина после успешного backend-вызова обновляет список и открывает следующий чат.

# Какие файлы были изменены
- `src/pages/MyQuestions.jsx`

# Какие ошибки были найдены
- Чат не позволял отправить файл без текста.
- Выбор файла был вынесен в отдельный неудобный блок.
- В СУП файлы задачи показывались вместе с проектными файлами.
- AI-история в проекте загружалась только для записей без `task_id`.
- После скрытия чата список в кабинете Бударина мог оставаться визуально несинхронизированным.

# Какие ошибки были исправлены
- File-only сообщения теперь проходят через существующую отправку сообщения и затем получают attachment.
- Удаление выбранного файла до отправки работает без потери текста.
- Файлы задач открываются и удаляются через существующий signed URL / soft delete механизм `AttachmentList`.
- AI-история проекта теперь включает записи задач и скрывает удалённые записи.
- Удаление AI-ответа вызывает существующую backend-функцию `delete-sup-ai-suggestion`.

# Какие проверки выполнены
- Проверка конфликтных маркеров: маркеры не найдены.
- Проверка сборки через `npm.cmd run build`.

# Что нужно проверить вручную
- Отправить сообщение с текстом и файлом в direct chat.
- Отправить сообщение только с файлом в direct chat.
- Отправить файл в чат с Будариным и проверить отображение в кабинете.
- Открыть и удалить свой файл через кнопку вложения.
- Прикрепить файл к конкретной задаче СУП.
- Открыть и удалить файл задачи.
- Удалить AI-ответ из истории Пользователь+ / СУП.
- Нажать “Скрыть” в кабинете Бударина и проверить исчезновение чата из списка.

# Риски и важные замечания
- SQL не изменялся. Для удаления вложений и AI-истории в Supabase должны быть выполнены ранее подготовленные SQL-файлы с `deleted_at` / `deleted_by`.
- Если bucket или RLS Storage не настроены, открытие файлов через signed URL может продолжить падать на стороне Supabase.
- Содержимое файлов в OpenAI не отправлялось.

# Следующий рекомендуемый шаг
Проверить в Supabase, что выполнены SQL-файлы для вложений, AI-history soft delete и скрытия чатов, затем вручную пройти сценарии отправки, открытия и удаления файлов.
