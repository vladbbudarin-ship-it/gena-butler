# Дерево доступа проектов

## 1. Название задачи
Добавить дерево доступа для проектов в модуле СУП.

## 2. Краткое описание задачи
Нужно было добавить проектное дерево подчинения, ручные доступы к задачам, импорт дерева из другого проекта и защитить контекст Гены СУП backend-фильтрацией по `project_id`.

## 3. Что было изменено
- Подготовлен SQL под реальные таблицы проекта: `sup_projects`, `sup_project_members`, `sup_tasks`.
- Добавлен backend-helper проверки проектного доступа.
- Добавлены Netlify Functions для загрузки дерева, смены руководителя, смены роли/видимости и импорта структуры.
- Добавлен экран “Структура проекта” во вкладку проектов.
- Обновлён Гена СУП: перед OpenAI-вызовом контекст фильтруется по текущему проекту и доступным задачам.
- Создание проектов/участников теперь пытается создавать строки дерева доступа, если SQL уже выполнен.

## 5. Какие ошибки были найдены
- В проекте используются таблицы `sup_*`, поэтому SQL из постановки нельзя было применять буквально к `projects/tasks`.
- У текущей видимости задач были только старые значения `project_public`, `assigned_only`, `custom`.
- Гена СУП не передавал список доступных задач проекта в контекст.

## 6. Какие ошибки были исправлены
- SQL сделан под реальные таблицы.
- Новые значения видимости задач добавлены с сохранением старых значений для совместимости.
- Backend-фильтрация доступа вынесена в общий helper.
- Добавлена защита от циклического подчинения.

## 7. Какие проверки выполнены
- Проверен синтаксис новых и изменённых Netlify Functions через `node --check`.
- Проверены конфликтные маркеры.
- Выполнена production-сборка Vite.

## 8. Результат npm run build
`npm.cmd run build` прошёл успешно.

## 9. Результат тестов, если они есть
E2E-тесты не запускались.

## 10. Что нужно проверить вручную
- Выполнить SQL `supabase/project-access-tree-schema.sql`.
- Открыть проект от имени администратора.
- Проверить вкладку “Структура”.
- Сменить руководителя и убедиться, что дерево обновляется.
- Попробовать создать цикл и получить ошибку.
- Импортировать дерево из другого проекта.
- Проверить видимость задач для разных участников.

## 11. Риски и важные замечания
- До выполнения SQL вкладка структуры не сможет сохранять дерево.
- Старые значения видимости задач сохранены для совместимости.
- Импорт копирует структуру, но не синхронизирует её постоянно.
