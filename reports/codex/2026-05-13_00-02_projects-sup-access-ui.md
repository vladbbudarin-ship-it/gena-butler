# Доработка страницы Проекты / СУП

## 1. Название задачи

Доработать интерфейс вкладки «Проекты», исправить видимость проектов, добавить удаление проекта и кабинет Пользователь+.

## 2. Краткое описание задачи

Нужно было привести страницу СУП к более понятной структуре, скрыть форму создания задачи по умолчанию, защитить удаление проекта, ограничить видимость проектов только участниками и владельцем, добавить подсказки ролей и отдельный кабинет Пользователь+.

## 3. Что было изменено

- На странице «Проекты» форма «Новая задача» скрыта по умолчанию.
- Добавлена красная кнопка «+ Новая задача» над списком задач.
- Добавлена кнопка «Отмена» и очистка формы задачи.
- Добавлена кнопка «Удалить проект» только для владельца системы или администратора проекта.
- Добавлена Netlify Function для защищённого удаления проекта.
- Загрузка списка проектов на frontend теперь дополнительно фильтрует проекты: не-owner видит только проекты, где он участник.
- Добавлен SQL-патч `supabase/project-access-fix.sql` для строгой RLS-видимости проектов и удаления проекта.
- Добавлены подсказки по ролям участников проекта.
- Добавлен экран «Кабинет Пользователь+» с ограниченным AI-помощником по доступным проектам.
- Навигация и профиль показывают «Кабинет Пользователь+» только для `user_plus` и `owner`.
- Улучшены стили страницы проектов и нового кабинета.

## 5. Какие ошибки были найдены

- Форма создания задачи была постоянно открыта.
- Не было отдельной backend-функции для удаления проекта.
- `user_plus` мог видеть лишние проекты, если RLS на базе была применена некорректно или остались старые политики.
- Участникам проекта не было понятно, чем отличаются роли.
- У `user_plus` не было отдельного ограниченного кабинета.

## 6. Какие ошибки были исправлены

- Создание задачи стало управляемым через скрываемую форму.
- Удаление проекта теперь проходит через backend с проверкой прав.
- Frontend больше не показывает не-owner проекты, где пользователь не состоит участником.
- Добавлены подсказки ролей.
- Добавлен отдельный кабинет Пользователь+ без owner-only функций.

## 10. Что нужно проверить вручную

- Выполнить `supabase/project-access-fix.sql` в Supabase SQL Editor.
- Под owner проверить, что видны все проекты и можно удалить проект.
- Под администратором проекта проверить удаление своего проекта.
- Под manager проверить, что удаления проекта нет, но можно создавать задачи.
- Под user_plus проверить, что чужие проекты не видны.
- Под обычным user, добавленным в проект, проверить, что виден только этот проект и доступные задачи.
- Проверить подсказки ролей на desktop и mobile.
- Проверить кабинет Пользователь+ под `user_plus` и owner.

## 11. Риски и важные замечания

- SQL-патч не выполнялся автоматически. Без выполнения `supabase/project-access-fix.sql` база может продолжить работать по старым политикам.
- Удаление проекта физическое, потому что подтверждение в задаче говорит: «Это действие нельзя отменить».
- Google Drive копия отчёта не создана автоматически из среды Codex.
- Telegram, чат, авторизация и OpenAI-логика не менялись, кроме уже существующей функции создания задачи, где восстановлена проверка `user_plus/owner`.

## 12. Следующий рекомендуемый шаг

Выполнить SQL из `supabase/project-access-fix.sql`, задеплоить изменения на Netlify и проверить сценарии owner / user_plus / user на опубликованном сайте.
