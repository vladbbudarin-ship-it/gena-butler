# Codex reports

Эта папка хранит markdown-отчёты о работе Codex над проектом «Дворецкий Гена».

Отчёты нужны, чтобы после каждой доработки, исправления ошибки или изменения логики оставалась понятная история:

- что именно менялось;
- какие файлы затронуты;
- какие ошибки были найдены и исправлены;
- какие проверки выполнены;
- что нужно проверить вручную перед публикацией.

## Где хранятся отчёты

Локально в проекте:

```text
reports/codex/
```

Копия на Google Drive:

```text
gena-dvoretskiy/reports/
```

Если Google Drive недоступен из среды Codex, отчёт всё равно должен быть сохранён локально, а в финальном ответе нужно явно написать, что файл требуется вручную перенести в Google Drive в папку `gena-dvoretskiy/reports/`.

## Формат имени файла

```text
YYYY-MM-DD_HH-mm_task-name.md
```

Пример:

```text
2026-05-11_18-30_fix-sup-tasks-rls.md
```

`task-name` должен быть коротким slug на латинице без секретов и приватных данных.

## Обязательные разделы отчёта

Каждый отчёт должен содержать:

1. Название задачи
2. Краткое описание задачи
3. Что было изменено
4. Какие файлы были изменены
5. Какие ошибки были найдены
6. Какие ошибки были исправлены
7. Какие проверки выполнены
8. Результат npm run build
9. Результат тестов, если они есть
10. Что нужно проверить вручную
11. Риски и важные замечания
12. Следующий рекомендуемый шаг

## Безопасность

В отчёты нельзя сохранять:

- API-ключи;
- токены;
- содержимое `.env`;
- Telegram token;
- Supabase service role key;
- OpenAI API key;
- любые приватные секреты.

В финальном сообщении после каждой задачи Codex должен указать:

- где сохранён локальный отчёт;
- где сохранена копия на Google Drive или почему она не сохранена;
- прошёл ли `npm run build`;
- есть ли незакоммиченные изменения.
